# Story 1.7: Admin Permission Configuration Interface

## Status
To Do

## Story
**As an** administrator,  
**I want** to easily configure user permissions for different agents,  
**so that** I can assign appropriate access levels to employees based on their job responsibilities.

## Problem Statement
With the new agent-based permission system, administrators need an intuitive interface to manage user permissions efficiently. The current user management interface lacks the capability to assign granular permissions for different agents, making it difficult to configure user access appropriately for their roles.

## Acceptance Criteria

1. **Permission Management Dashboard:** Centralized interface for viewing and managing all user permissions across agents
2. **User Permission Matrix:** Visual matrix showing users vs agents with their current permission levels
3. **Individual User Permission Dialog:** Detailed permission configuration interface for specific users
4. **Bulk Permission Assignment:** Ability to assign the same permissions to multiple users simultaneously
5. **Permission Templates:** Comprehensive template management system with pre-defined system templates (Client Specialist, Report Analyst, Document Processor, Audio Specialist) and custom template creation, enabling administrators to standardize permission patterns, reduce configuration time from 30 minutes to 2 minutes per user, and ensure consistent access control across similar job roles
6. **Real-time Permission Updates:** Changes take effect immediately without requiring user logout/login
7. **Permission Change Audit:** Complete history of permission changes with administrator details
8. **Permission Impact Warnings:** Clear warnings when removing permissions that might affect user's current work

## Dev Notes

### Architecture Insights from Sprint Change Proposal

The admin permission configuration interface is the critical management layer that makes the enhanced permission system usable. It transforms complex permission data into an intuitive administrative experience.

**Key Design Principles:**
- **KISS Approach:** Simple, clear interface avoiding over-complication
- **YAGNI Implementation:** Build only essential features, avoid advanced role templates initially
- **Visual Clarity:** Matrix view provides immediate overview of permission landscape
- **Efficient Bulk Operations:** Minimize administrative overhead for large user bases

interface PermissionMatrixData {

// Matrix cell component
const PermissionCell: React.FC<{
  user: User;
  agent: string;
  permissions: AgentPermissions;
  onToggle: (user_id: string, agent: string, operation: string) => void;
}> = ({ user, agent, permissions, onToggle }) => {
  const hasAnyPermission = Object.values(permissions).some(Boolean);
  
  return (
    <div className={`permission-cell ${hasAnyPermission ? 'has-permissions' : 'no-permissions'}`}>
      <DropdownMenu>
        <DropdownMenuTrigger asChild>
          <Button variant="ghost" size="sm">
            {hasAnyPermission ? <CheckCircle /> : <Circle />}
          </Button>
        </DropdownMenuTrigger>
        <DropdownMenuContent>
          {AGENT_OPERATIONS.map(operation => (

**Individual User Permission Dialog:**
```typescript
const UserPermissionsDialog: React.FC<{
  user: User;
  isOpen: boolean;
  onClose: () => void;
}> = ({ user, isOpen, onClose }) => {
  const { data: permissions, mutate: updatePermissions } = useUserPermissions(user.user_id);
  const [localPermissions, setLocalPermissions] = useState(permissions);
  const [hasChanges, setHasChanges] = useState(false);

  const handlePermissionToggle = (agent: string, operation: string) => {
    const newPermissions = {
      ...localPermissions,
      [agent]: {
        ...localPermissions[agent],
        [operation]: !localPermissions[agent]?.[operation]
      }
    };
    
    setLocalPermissions(newPermissions);
    setHasChanges(true);
  };

  const handleSave = async () => {
    await updatePermissions.mutateAsync({
      user_id: user.user_id,
      permissions: localPermissions
    });
    
    toast.success('Permissões atualizadas com sucesso');
    setHasChanges(false);
    onClose();
  };

  return (
    <Dialog open={isOpen} onOpenChange={onClose}>
      <DialogContent className="max-w-4xl">
        <DialogHeader>
          <DialogTitle>Gerenciar Permissões - {user.full_name}</DialogTitle>
          <DialogDescription>
            Configure as permissões do usuário para cada agente do sistema
          </DialogDescription>
        </DialogHeader>
        
        <Tabs defaultValue="client_management">
          {AGENTS.map(agent => (
            <TabsContent key={agent.name} value={agent.name}>
              <Card>
                <CardHeader>
                  <CardTitle>{agent.display_name}</CardTitle>
                  <CardDescription>{agent.description}</CardDescription>
                </CardHeader>
                <CardContent>
                  <div className="grid grid-cols-2 gap-4">
                    {agent.operations.map(operation => (
                      <div key={operation} className="flex items-center space-x-2">
                        <Checkbox
                          id={`${agent.name}-${operation}`}
                          checked={localPermissions[agent.name]?.[operation] || false}
                          onCheckedChange={() => handlePermissionToggle(agent.name, operation)}
                        />
                        <Label htmlFor={`${agent.name}-${operation}`}>
                          {OPERATION_LABELS[operation]}
                        </Label>
                      </div>
                    ))}
                  </div>
                </CardContent>
              </Card>
            </TabsContent>
          ))}
        </Tabs>
        
        <DialogFooter>
          <Button variant="outline" onClick={onClose}>
            Cancelar
          </Button>
          <Button onClick={handleSave} disabled={!hasChanges}>
            Salvar Alterações
          </Button>
        </DialogFooter>
      </DialogContent>
    </Dialog>
  );
};
```

@router.put("/permissions/bulk")
@require_role("admin")
async def bulk_update_permissions(
    bulk_request: BulkPermissionRequest,
    current_user: User = Depends(get_current_user)
):
    """Apply permission changes to multiple users simultaneously."""
    results = []
    
    for user_id in bulk_request.user_ids:
        try:
            await permission_service.update_user_permissions(
                user_id=user_id,
                permissions=bulk_request.permissions,
                updated_by=current_user.user_id
            )
            results.append({"user_id": user_id, "status": "success"})
        except Exception as e:
            results.append({"user_id": user_id, "status": "error", "error": str(e)})
    
    return {"results": results}

### Real-time Updates Implementation

**WebSocket Permission Updates:**
```python
# WebSocket manager for real-time updates
class PermissionUpdateManager:
    def __init__(self):
        self.active_connections: List[WebSocket] = []
    
    async def connect(self, websocket: WebSocket):
        await websocket.accept()
        self.active_connections.append(websocket)
    
    def disconnect(self, websocket: WebSocket):
        self.active_connections.remove(websocket)
    
    async def broadcast_permission_update(self, user_id: UUID, permissions: dict):
        message = {
            "type": "permission_update",
            "user_id": str(user_id),
            "permissions": permissions,
            "timestamp": datetime.utcnow().isoformat()
        }
        
        for connection in self.active_connections:
            try:
                await connection.send_json(message)
            except ConnectionClosedOK:
                self.disconnect(connection)

permission_manager = PermissionUpdateManager()

@app.websocket("/ws/permissions")
async def websocket_endpoint(websocket: WebSocket):
    await permission_manager.connect(websocket)
    try:
        while True:
            await websocket.receive_text()
    except WebSocketDisconnect:
        permission_manager.disconnect(websocket)
```

**Frontend WebSocket Integration:**
```typescript
// Permission update hook with WebSocket
export const usePermissionUpdates = () => {
  const queryClient = useQueryClient();
  const [socket, setSocket] = useState<WebSocket | null>(null);

  useEffect(() => {
    const ws = new WebSocket('ws://localhost:8000/ws/permissions');
    
    ws.onmessage = (event) => {
      const data = JSON.parse(event.data);
      
      if (data.type === 'permission_update') {
        // Invalidate permission queries for updated user
        queryClient.invalidateQueries(['user-permissions', data.user_id]);
        queryClient.invalidateQueries(['permission-matrix']);
        
        // Show notification if it affects current user
        const currentUser = queryClient.getQueryData(['current-user']);
        if (currentUser?.user_id === data.user_id) {
          toast.info('Suas permissões foram atualizadas');
        }
      }
    };

    setSocket(ws);
    
    return () => {
      ws.close();
    };
  }, [queryClient]);

  return socket;
};
```
