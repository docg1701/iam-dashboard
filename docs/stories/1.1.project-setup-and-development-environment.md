### Project Structure Requirements
**[Source: architecture/unified-project-structure.md]**

**Monorepo Structure:**
```
multi-agent-iam-dashboard/
├── apps/
│   ├── frontend/          # Next.js 15 with App Router
│   └── backend/           # FastAPI with UV management
├── packages/
│   ├── shared/            # Shared types and utilities
│   ├── ui/                # Shared UI components

├── infrastructure/        # Terraform + Ansible
```

**Root Package.json Configuration:**
- Workspaces: `["apps/*", "packages/*"]`
- Scripts for unified dev, build, test, lint commands
- Concurrency for running frontend and backend simultaneously

### Coding Standards
**[Source: architecture/coding-standards.md]**

**Critical Rules:**
- Maximum 500 lines per file, functions under 50 lines
- 80% minimum code coverage - NO EXCEPTIONS
- Type Sharing: Always define shared types in `shared/types/`
- Environment Variables: Access only through config objects in `core/config.py`
- File Organization: Co-located tests in `__tests__` folders
- Language Consistency: All code and technical content in English

**Naming Conventions:**
- Database Tables: `snake_case`
- API Routes: `kebab-case`
- Components: `PascalCase`
- Hooks: `camelCase with 'use'`
- Environment Variables: `UPPER_SNAKE_CASE`

### Database Configuration
**[Source: architecture/data-models.md, architecture/backend-architecture.md]**

**SQLModel Integration Pattern:**
```python
class ClientBase(SQLModel):
    full_name: str = Field(min_length=2, max_length=255)
    ssn: str = Field(regex=r'^\d{3}-\d{2}-\d{4}$')
    birth_date: date
    status: ClientStatus = ClientStatus.ACTIVE

class Client(BaseModel, ClientBase, table=True):
    __tablename__ = "clients"
    client_id: UUID = Field(primary_key=True, alias="id")
    created_by: UUID = Field(foreign_key="users.user_id")
```

**Required Extensions:**
- pgvector extension for future RAG processing
- Proper indexing strategy for foreign keys and search fields
- Alembic migrations with version control

**[Source: architecture/backend-architecture.md]**

**JWT Authentication Flow:**
- 15-minute access tokens with refresh token mechanism
- Role-based access control: sysadmin, admin, user

### Testing
**[Source: architecture/testing-strategy.md]**

**Test File Locations:**
- Frontend tests: `apps/frontend/tests/` with co-located `__tests__/` folders
- Backend tests: `apps/backend/src/tests/` organized by type
- E2E tests: `tests/playwright/` for critical workflows

**Testing Standards:**
- Minimum 80% code coverage requirement across all modules
- Test-Driven Development when possible
- Factory patterns for test data generation
- Co-located tests in `__tests__/` folders next to code

**Testing Frameworks and Patterns:**
- **Backend:** pytest >=8.4.1 with pytest-asyncio >=1.1.0 for API and business logic
- **Frontend:** Vitest >=3.2.4 with Testing Library >=16.3.0 for unit tests
- **E2E:** Playwright >=1.54 for critical user workflows and agent interactions
