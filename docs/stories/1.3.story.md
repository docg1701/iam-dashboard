**FastAPI Authorization Patterns** [Source: architecture/api-design-and-agent-integration.md#5.1]:

```python
@router.post("/agents", status_code=status.HTTP_201_CREATED)
async def create_agent(
    agent_request: AgentCreateRequest, 
    current_user: dict = Depends(AuthManager.get_current_user)
):
    if not AuthManager.check_permission(current_user, "agent_management", "create"):
        raise HTTPException(status_code=403, detail="Insufficient permissions")
```

### Component Specifications
**Current Authorization Issue** [Source: app/ui_components/admin_dashboard.py]:
The admin dashboard currently checks for roles using string comparison:
```python
user_role = current_user.get("role", "")
if user_role not in ["admin_user", "sysadmin"]:  # PROBLEM: String values don't match enum
```

But UserRole enum uses different values:
- UserRole.SYSADMIN.value = "sysadmin" ✓ (matches)
- UserRole.ADMIN_USER.value = "admin_user" ✓ (matches)

**Issue Analysis**: The role checking logic appears correct, but there may be issues with:
1. How roles are stored in the session (AuthManager.login_user)
2. How roles are retrieved from the database
3. Enum value serialization/deserialization
