### Data Models
**Missing Questionnaire Drafts Table** [Source: architecture/data-models-and-schema-integration.md#3.3]:
```sql
CREATE TABLE IF NOT EXISTS questionnaire_drafts (
    draft_id SERIAL PRIMARY KEY,
    client_id INTEGER NOT NULL REFERENCES clients(client_id) ON DELETE CASCADE,
    user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    template_type VARCHAR(100) NOT NULL,
    draft_content JSONB NOT NULL,
    status VARCHAR(50) DEFAULT 'draft',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_questionnaire_drafts_client_id 
ON questionnaire_drafts(client_id);

CREATE INDEX IF NOT EXISTS idx_questionnaire_drafts_user_id 
ON questionnaire_drafts(user_id);
```

### Technical Constraints
**Database Technology** [Source: architecture/tech-stack.md]:
- PostgreSQL >= 14 with ACID compliance
- SQLAlchemy >= 2.0.41 with async support
- Alembic >= 1.13.0 for schema versioning
- pgvector >= 0.5.0 extension for vector storage

**Schema Standards** [Source: architecture/coding-standards.md#database-naming-conventions]:
- Tables: snake_case (plural) - `questionnaire_drafts`
- Primary Keys: `{entity}_id` - `draft_id`
- Foreign Keys: `{referenced_entity}_id` - `client_id`, `user_id`
- Timestamps: `{action}_at` - `created_at`, `updated_at`
- Booleans: `is_{state}` - status field uses VARCHAR for draft/published states
- Indexes: `idx_{table}_{column}` - `idx_questionnaire_drafts_client_id`
